# Детекция занятости столиков по видео

Прототип системы детекции уборки/занятости столиков на основе компьютерного зрения.

## 📋 Требования

- Python 3.8+
- OpenCV
- Pandas
- Ultralytics (YOLOv8)
- NumPy

## 🚀 Установка

```bash
# Установка зависимостей
pip install -r requirements.txt
```

## 🎬 Запуск

```bash
python main.py --video <путь_к_видео>
```

**Пример:**
```bash
python main.py --video "видео 1.mp4" --output "output.mp4"
```

### Управление
- **Q** — завершить обработку
- **Закрытие окна** — завершить обработку

## 📊 Какое видео и столик выбраны

| Параметр | Значение |
|----------|----------|
| **Видео** | `видео 1.mp4` (укажите название вашего видео) |
| **Столик** | Выделяется вручную через `cv2.selectROI` при запуске |

**Инструкция по выбору столика:**
1. При запуске откроется окно с первым кадром видео
2. Выделите прямоугольную область вокруг столика
3. Нажмите **ENTER**, затем **SPACE** для подтверждения
4. Координаты ROI будут выведены в консоль

## 🔍 Логика детекции событий

### Используемые технологии
- **YOLOv8n** (Ultralytics) — детекция людей в ROI
- **TableTracker** — машина состояний для отслеживания событий

### Состояния столика

| Состояние | Описание | Цвет рамки |
|-----------|----------|------------|
| `EMPTY` | Стол пустой (нет людей в ROI) | 🟢 Зелёный |
| `OCCUPIED` | Стол занят (есть люди в ROI) | 🔴 Красный |
| `APPROACHING` | Подход к столу (появление после пустоты) | 🟡 Жёлтый |

### Алгоритм работы

```
1. Пользователь выделяет ROI столика
2. Для каждого кадра:
   └─> Каждые 2 кадра: YOLO детектирует людей в ROI
   └─> Если есть люди (conf > 0.5) → стол ЗАНЯТ (Occupied)
   └─> Если нет людей → стол ПУСТОЙ (Empty)
3. TableTracker фиксирует переходы между состояниями
4. События записываются с временными метками
```

### Оптимизация
- Детекция YOLO выполняется **каждые 2 кадра** (экономия ~50% времени)
- Между детекциями используются закэшированные данные

## 🖼️ Пример проблемного кадра

### Проблема: Ложное срабатывание при частичном перекрытии

**Описание проблемы:**
Когда человек проходит мимо столика, не садясь, YOLO может детектировать его в ROI, что вызывает ложное событие `OCCUPIED`.

**Скриншот:**
![Проблемный кадр](/screenshots/Screenshot_2026-03-29_222827.png)

*На скриншоте видно, что человек находится рядом со столиком, но не сидит за ним.*

### Возможные улучшения
1. Добавить трекинг людей (DeepSORT) для отслеживания траекторий
2. Разделить ROI на зоны "сидения" и "прохода"
3. Использовать позы (OpenPose) для определения факта сидения

## 🔧 Настройки (можно изменить в коде)

| Параметр | Значение | Описание |
|----------|----------|----------|
| `DETECT_EVERY_N_FRAMES` | 2 | Частота детекции YOLO |
| `CONFIDENCE_THRESHOLD` | 0.5 | Порог уверенности YOLO |
| `empty_threshold_sec` | 2.0 | Время пустоты для триггера подхода |

## 📝 Примечания

- Скрипт выводит статистику **после окончания видео** или при закрытии окна
- После окончания видео или после закрытия окна **формируется файл** с названием, указаным при запуске (по ум. output.mp4)
- Для ускорения обработки можно увеличить `DETECT_EVERY_N_FRAMES`
- Для повышения точности можно уменьшить порог детекции
